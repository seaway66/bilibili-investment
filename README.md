# bilibili-investment

B站投资财经类视频深度分析工具 — 专为股票/行业/财务分析视频设计，通过多模态 AI 还原 UP 主研究框架，生成专业投研报告。

## 原理

本工具在 [bilibili-summary](../bilibili-summary/README.md) 的通用流水线基础上，针对**投资研究场景**进行了深度定制。核心差异在于视觉识别的 Prompt 工程和报告生成逻辑。

```
B站投资视频(BV号)
  │
  ├─ 1. 视频获取 ────────────────────────────────────────────
  │   调用 B站公开 API 获取视频元数据
  │   使用 you-get / yt-dlp 下载视频+音频，失败时回退到直接 API 下载
  │
  ├─ 2. 关键帧提取 ────────────────────────────────────────────
  │   每 18 秒截取一帧，通过 OpenCV pixel-diff 去重
  │   对含有图表的画面（K线图、财务图、产业链图）额外保留
  │
  ├─ 3. 投资视角视觉识别 ────────────────────────────────────
  │   使用投资专用 Prompt 逐帧分析：
  │   · 列出画面中所有数字、百分比、金额
  │   · 识别公司名称、股票代码、行业术语
  │   · 描述图表趋势、数据对比关系
  │   · 提取产业链结构和公司卡位信息
  │
  │   质量管理：自动检测输出质量问题（<50字/数据缺失/矛盾）
  │   满足任一条件 → 自动回退 qwen-vl-max 重新识别
  │
  ├─ 4. 语音转录 ────────────────────────────────────────────
  │   Whisper tiny 中文语音转文字，输出纯文本 + 时间戳
  │
  └─ 5. 投研报告生成 ────────────────────────────────────────
       AI 以投资分析师视角综合画面数据 + 音频论述：
       · 还原 UP 主的投资 Thesis 和逻辑链
       · 汇总市场/财务/参数三张数据表
       · 逐一分析涉及标的并给出评级
       · 识别风险点和后续关注信号
       · 强制附加投资免责声明
```

### 核心架构思想：感知与分析分离

本工具最关键的架构设计是**将"看懂画面"和"分析投资逻辑"两个任务分离到不同层级的模型**。这一点对于投资场景尤为重要——视频中的财务图表、K 线走势、产业链图谱需要被准确识别，但对它们的深度解读（估值合理性、竞争壁垒、风险判断）需要极强的推理能力。

```
┌─────────────────────────────────────────────────────────────┐
│  感知层（低成本多模态模型）                                    │
│  GLM-4.6V-Flash（免费）/ qwen-vl-max（付费兜底）             │
│  任务：逐帧识别数字、公司名、图表趋势、产业链结构               │
│  要求：能"看见并提取"画面中的信息即可                         │
│  成本：免费 ~ 极低                                           │
├─────────────────────────────────────────────────────────────┤
│  分析层（高质量纯文本模型）                                    │
│  Claude / DeepSeek 等                                       │
│  任务：综合所有帧数据 + 音频转录 → 还原投资逻辑、生成投研报告   │
│  要求：强推理、投资知识、逻辑链分析、结构化输出                 │
│  成本：仅一次调用，可控                                       │
└─────────────────────────────────────────────────────────────┘
```

**为什么投资分析场景更需要这种架构？**

如果直接用 GPT-4o 或 Gemini 分析投资视频，面临三重困境：

1. **成本爆炸**：投资类视频通常 15-40 分钟，提取 50-130 帧，每帧都包含复杂图表需要仔细识别。用高端多模态模型逐帧处理，单次分析成本可能高达数美元甚至更多
2. **精度陷阱**：高端多模态模型虽然"聪明"，但对于"从图表中精准读出每一个数字"这类机械性任务，反而未必比专门优化过的轻量模型更准确——就像让一个基金经理去手工录入财报数据，既浪费又不一定可靠
3. **分析深度不足**：即使勉强把所有帧塞进 context window，多模态模型在同时处理视觉信息和做深度投资推理时，推理质量往往不如纯文本模型在充裕的文本 context 中专注分析

本工具的方案：

1. **感知层**用免费/低成本多模态模型逐帧提取原始数据——数字、公司名、图表描述、产业链关系。这是机械性的"数据录入"工作，免费模型完全胜任；遇到复杂图表自动回退 qwen-vl-max 确保不遗漏关键数据
2. **分析层**用最强大的纯文本推理模型（如 Claude），一次性接收所有结构化数据 + 音频转录，以投资分析师的角色深度工作：还原逻辑链、评估数据可信度、识别风险、给出评级

| | 传统方案（高端多模态直接分析） | 本工具（感知-分析分离） |
|---|---|---|
| **视觉模型成本** | GPT-4o/Gemini 逐帧处理，极贵 | GLM 免费 + 仅异常帧回退付费模型 |
| **分析模型成本** | 含在多模态调用中 | 仅一次纯文本调用 |
| **可处理帧数** | 受 context window 严格限制 | 感知层无限制，长视频轻松覆盖 |
| **数据提取精度** | 高智商但可能"粗心" | 轻量模型专注提取，异常自动兜底 |
| **投资推理深度** | 图文混杂处理，推理分散 | 纯文本充裕空间，推理专注深入 |
| **模型选择自由度** | 绑定单一多模态模型 | 感知/分析独立选型，各自升级 |

### 与 bilibili-summary 的关键差异

| 维度 | bilibili-summary | bilibili-investment |
|------|:---|:---|
| **识图 Prompt** | "描述画面内容" | "列出所有数字、公司名、图表趋势、产业链结构" |
| **分析视角** | 通用内容理解 | 投资分析师视角 |
| **数据要求** | 定性描述为主 | 定量数据必须，缺失则标记 |
| **回退策略** | 输出 < 50 字触发 | 输出 < 50 字 / 数值缺失 / 数据矛盾 / 核心数据帧识别不清 |
| **报告结构** | 5 章节通用模板 | 9 章节投研专业模板 |
| **免责声明** | 无 | 必须（投资风险警告） |

### 视觉识别质量保障

本工具对视觉识别的质量要求远高于通用总结。针对投资场景中关键的**财务图表**和**产业链图**，设计了多层质量保障：

```
GLM-4.6V-Flash 识别
  ├─ 检查1：输出字数 ≥ 50？
  │   └─ 否 → 换详细 prompt 重试
  ├─ 检查2：包含数字/百分比？
  │   └─ 否（核心数据帧）→ 回退 qwen-vl-max
  ├─ 检查3：数据与其他帧矛盾？
  │   └─ 是 → 回退 qwen-vl-max 重新识别
  └─ 检查4：API 429 限流？
      └─ 是 → 等待 5 秒重试，仍失败则回退 qwen-vl-max
```

### 核心功能

|------|------|
| 📥 **视频获取** | you-get / yt-dlp / B站 API 三层下载保障 |

| **3. 行业/技术分析** | 行业背景、竞争壁垒、产业链结构、公司卡位 |

| **6. UP 主研究方法论** | 信息来源、分析框架、判断强度 |
| **7. 风险与不确定性** | UP 主承认的风险 + AI 发现的潜在风险 |
| **8. 后续关注点** | 催化剂事件、跟踪指标、验证信号 |
| **9. 免责声明** | 投资风险警告（强制包含） |

## 特点

### 🧠 感知与分析分离 — 核心架构优势

这是本工具与"直接用 GPT-4o/Gemini 分析投资视频"方案最本质的区别，也是投资分析场景下最重要的架构决策。

**传统方案的三重困境**：

1. **成本不可控**：投资视频通常 15-40 分钟，提取 50-130 帧，每帧用高端多模态模型处理，单次分析轻易突破数美元。对于日常跟踪多个 UP 主的投资者来说完全不现实
2. **精度不可靠**：高端模型"聪明但可能粗心"——在"从复杂图表中逐一读出数字"这种机械性任务上，GPT-4o 未必比专门优化过的轻量模型更准确。而投资场景中，一个数字读错（如净利润 50 亿读成 15 亿）就可能导致完全错误的投资判断
3. **分析不够深**：多模态模型需要同时处理图片和文字，context window 被大量图片占据，留给深度投资推理的空间不足

**本工具的方案**：

- **感知层**：免费多模态模型（GLM-4.6V-Flash）专注做"数据录入"——从每帧画面中提取数字、公司名、图表描述。这是机械性工作，免费模型完全胜任；仅在复杂图表识别不清时自动回退 qwen-vl-max
- **分析层**：最强大的纯文本推理模型（如 Claude）专注做"投资分析"——还原逻辑链、评估估值、识别风险、给出评级。这是深度推理工作，需要最强的文本理解能力，完全不依赖多模态

**结果**：视觉识别近乎零成本（仅异常帧触发付费回退），分析质量不打折（纯文本模型在充裕 context 中专注推理），视频长度不再受限，两个模型可以独立升级。对于需要持续跟踪多位投资 UP 主的用户，这是唯一经济可行的方案。

### 🎯 投资专用 Prompt 工程

区别于通用总结的"描述画面"，本工具使用投资分析视角的专用 Prompt，引导视觉模型主动提取：

- 画面中的**所有数字**（金额、百分比、增长率、PE、PB 等）
- **公司名称和股票代码**
- **图表趋势**（上升/下降/震荡、支撑位/压力位）
- **产业链位置**（上游/中游/下游、供应商/客户关系）

这使得 AI 能够从画面中提取投资研究所需的定量信息，而非仅仅"这段在讲新能源"。

### 🛡️ 四级质量保障机制

投资数据容错率极低。普通视频画面识别错了可能无关紧要，但财务数据（如"净利润 50 亿"误识别为"15 亿"）会导致完全错误的投资判断。本工具设置了四级质量检查：

1. **字数检查**：输出 < 50 字视为识别不充分
2. **数据完整性检查**：核心数据帧（财务图表）必须包含数值信息
3. **一致性检查**：同一数据点在不同帧中的数值应一致
4. **API 健康检查**：429 限流自动等待重试

任一检查不通过 → 自动回退到更精确的 qwen-vl-max 模型。

### 📊 强制数据结构化

不同于通用总结的可选表格，本工具**必须**输出三张结构化数据表：

- **市场数据表**：市值、PE、PB、涨跌幅、成交量等
- **财务数据表**：营收、利润、毛利率、ROE、增速等
- **关键参数表**：行业特定指标（如新能源的装机量、半导体的制程节点等）

缺失的数据会明确标注 `[数据缺失]`，而非静默跳过，确保报告的完整性和可审计性。

### 🏷️ 标的评级体系

对视频中涉及的每家公司/标的，报告会给出标准化评级：

- 🥇 **核心推荐** — UP 主最看好的标的
- 🥈 **重点关注** — 有亮点但存在不确定性的标的
- 🥉 **跟踪观察** — 产业链相关但非直接推荐

每个标的附带：产业链位置、财务亮点、风险点和评级理由。

### 🔍 逻辑链完整还原

不仅记录"UP 主说了什么"，更通过现象→推论→结论→标的的四层逻辑框架，完整还原 UP 主的投资分析过程。这让读者可以审视每一步推论的合理性，而非被动接受结论。

### ⚠️ 强制风险提示

报告末尾强制包含投资免责声明，并区分两类风险：
- **UP 主承认的风险**：视频中 UP 主自己提到的风险因素
- **AI 发现的风险**：基于数据异常、逻辑漏洞或行业知识的额外风险提示

### 💰 默认免费可用

与 bilibili-summary 一样，默认使用 GLM-4.6V-Flash 免费模型 + 本地 Whisper tiny，日常使用零费用。仅在遇到复杂财务图表时自动切换付费模型，确保关键数据的准确性。

### 🤖 AI Agent 友好

作为 Claude Code Skill 封装，可通过自然语言调用：

```

```

AI Agent 会自动执行流水线，以投资分析师视角处理每一帧，生成专业投研报告。

## 环境要求

- Python 3.10+

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 API Key
# Windows PowerShell:
$env:GLM_API_KEY = "your_glm_key"
# （可选）付费兜底模型：
$env:DASHSCOPE_API_KEY = "your_qwen_key"
# Linux/Mac:
export GLM_API_KEY="your_glm_key"
export DASHSCOPE_API_KEY="your_qwen_key"

# 3. 生成投研报告
python scripts/pipeline.py BV1DoTx6yEev -o ./output

# 4. 图表密集的视频直接用千问（更精准）
python scripts/pipeline.py BV1DoTx6yEev --vision-provider qwen
```

## 配置说明

| 环境变量 | 说明 | 必填 |
|---------|------|:---:|
| `GLM_API_KEY` | 智谱 API Key（免费视觉模型，推荐） | 是 |
| `DASHSCOPE_API_KEY` | 千问 API Key（付费兜底，图表密集视频推荐配置） | 推荐 |
| `OPENAI_API_KEY` | OpenAI API Key | 否 |
| `DOUBAO_API_KEY` | 豆包 API Key | 否 |
| `VISION_PROVIDER` | 默认视觉供应商 (glm/qwen/openai/doubao) | 否（默认 glm） |
| `VISION_MAX_TOKENS` | 视觉模型最大输出 tokens（投资报告建议 4096+） | 否（默认 4096） |
